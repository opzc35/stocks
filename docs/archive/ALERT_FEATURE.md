# 价格预警功能详细说明

## 概述

价格预警系统让你可以设置价格目标，当市场价格达到目标时自动通知你，帮助你及时捕捉交易机会，不错过关键价位。

## 功能特性

### 1. 🔔 灵活的预警设置

**支持两种预警类型：**

- **突破预警 (▲)**：当价格上涨并达到或超过目标价格时触发
  - 适用场景：突破阻力位、止盈提醒、趋势确认
  
- **跌破预警 (▼)**：当价格下跌并达到或低于目标价格时触发
  - 适用场景：跌破支撑位、买入机会、止损提醒

**设置步骤：**
1. 选择预警类型（突破/跌破）
2. 输入目标价格
3. 点击"添加"按钮
4. 预警立即生效，开始监控

### 2. 🚨 实时触发通知

当价格达到预警条件时，系统会：

**视觉通知：**
- 右上角弹出红色预警通知框
- 🚨 图标闪烁提醒
- 显示触发的预警信息
- 显示当前价格

**音频通知：**
- 播放提示音（浏览器允许时）
- 即使切换到其他标签也能听到

**通知选项：**
- **知道了**：关闭通知，预警标记为已触发
- **查看详情**：跳转到预警面板查看详情

**自动消失：**
- 通知会在 10 秒后自动消失
- 可以手动点击关闭

### 3. 📊 预警状态监控

**预警面板实时显示：**

- **当前价格**：实时显示，方便对比目标价格
- **预警列表**：所有设置的预警，按创建时间排序
- **状态指示**：
  - 绿点 + "等待中" = 预警未触发
  - 红点 + "已触发" = 预警已触发
- **视觉区分**：
  - 突破预警：左侧绿色边框
  - 跌破预警：左侧橙色边框
  - 已触发预警：红色边框 + 脉动动画

### 4. 🔄 预警管理

**重置预警：**
- 点击 🔄 按钮重置已触发的预警
- 预警恢复到"等待中"状态
- 可以继续监控相同价格

**删除预警：**
- 点击 🗑️ 按钮删除预警
- 适用于不再需要的预警

**批量管理：**
- 支持同时设置多个预警
- 每个预警独立触发
- 互不影响

## 使用场景示例

### 场景 1：突破买入策略

**目标：** BTC 突破 70,000 美元时买入

**操作：**
1. 选择"突破 ▲"
2. 输入目标价格：70000
3. 添加预警
4. 等待触发通知
5. 收到通知后及时下单

### 场景 2：止损保护

**目标：** 持有 BTC，价格跌破 65,000 美元时止损

**操作：**
1. 选择"跌破 ▼"
2. 输入目标价格：65000
3. 添加预警
4. 价格触碰 65,000 时收到通知
5. 及时卖出止损

### 场景 3：多点位布局

**目标：** 在多个关键价位设置预警

**操作：**
1. 添加突破预警：68000（阻力位1）
2. 添加突破预警：70000（阻力位2）
3. 添加跌破预警：65000（支撑位1）
4. 添加跌破预警：62000（支撑位2）
5. 系统同时监控所有价位
6. 任意价位触发都会收到通知

### 场景 4：波段交易

**目标：** 在价格区间内高抛低吸

**操作：**
1. 设置跌破预警 64000（买入区间）
2. 买入后，设置突破预警 68000（卖出区间）
3. 卖出后，重置第一个预警继续监控
4. 重复操作，波段套利

## 技术实现

### 预警检测机制

```typescript
// 每当价格更新时自动检查所有预警
useEffect(() => {
  alerts.forEach(alert => {
    if (alert.isTriggered) return // 跳过已触发的预警

    const shouldTrigger =
      (alert.condition === 'above' && currentPrice >= alert.targetPrice) ||
      (alert.condition === 'below' && currentPrice <= alert.targetPrice)

    if (shouldTrigger) {
      // 触发预警通知
      triggerAlertNotification(alert)
    }
  })
}, [currentPrice, alerts])
```

### 数据结构

```typescript
interface PriceAlert {
  id: string                    // 唯一标识
  symbol: string                // 交易对
  condition: 'above' | 'below'  // 预警条件
  targetPrice: number           // 目标价格
  currentPrice?: number         // 触发时的价格
  isTriggered: boolean          // 是否已触发
  createdAt: number             // 创建时间
  triggeredAt?: number          // 触发时间
  message?: string              // 预警消息
}
```

## 最佳实践

### 1. 合理设置预警数量
- 建议同时不超过 10 个预警
- 太多预警会频繁打断
- 专注关键价位

### 2. 及时管理已触发预警
- 定期清理已触发的预警
- 或重置后继续使用
- 保持预警列表简洁

### 3. 结合图表使用
- 在图表上确定关键价位
- 设置预警前查看支撑/阻力
- 参考技术指标

### 4. 测试通知功能
- 首次使用时设置测试预警
- 确认通知能正常收到
- 检查浏览器通知权限

### 5. 价格留余地
- 突破预警设置略高于阻力位
- 跌破预警设置略低于支撑位
- 避免假突破触发

## 故障排除

### 通知没有弹出？
1. 检查浏览器是否允许通知
2. 确认预警条件设置正确
3. 查看当前价格是否已触发

### 音频没有播放？
1. 浏览器可能阻止自动播放
2. 需要用户交互后才能播放
3. 检查浏览器音量设置

### 预警不触发？
1. 确认价格数据正常更新
2. 检查预警条件是否合理
3. 查看预警是否已被触发过

### 触发后立即消失？
1. 通知默认 10 秒后消失
2. 可以手动点击"知道了"关闭
3. 预警会保留在列表中

}
