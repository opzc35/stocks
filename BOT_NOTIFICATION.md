# 社交媒体机器人通知功能

## 概述

社交媒体机器人功能让你可以将价格预警自动推送到 Telegram、Discord、Slack、企业微信等平台，无论你在哪里都能第一时间收到通知。

## 支持的平台

### 1. 📱 Telegram
- **优势**：免费、快速、支持全球大部分国家访问
- **需要**：Bot Token 和 Chat ID
- **适合**：个人使用、小团队

### 2. 💬 Discord
- **优势**：游戏社区流行、支持富文本
- **需要**：Webhook URL
- **适合**：游戏玩家、技术社区

### 3. 💼 Slack
- **优势**：企业级、集成丰富
- **需要**：Webhook URL
- **适合**：工作团队、企业用户

### 4. 💚 企业微信
- **优势**：国内流行、企业认可
- **需要**：Webhook URL
- **适合**：国内企业、团队协作

## 快速开始

### 步骤 1：打开机器人设置

1. 点击顶部导航栏的 **"🤖 机器人"** 按钮
2. 机器人设置面板会弹出

### 步骤 2：添加机器人

1. 点击 **"+ 添加机器人"** 按钮
2. 选择平台（Telegram、Discord、Slack、企业微信）
3. 填写机器人信息：
   - **机器人名称**：给机器人起个名字（例如：交易预警机器人）
   - **平台配置**：根据选择的平台填写相应信息

### 步骤 3：测试连接

1. 点击 **"🧪 测试"** 按钮
2. 查看测试结果：
   - ✅ 测试成功：配置正确
   - ❌ 测试失败：检查配置信息

### 步骤 4：启用机器人

1. 使用右侧开关启用机器人
2. 绿色 = 已启用，灰色 = 已禁用
3. 只有启用的机器人才会发送通知

## 平台配置指南

### 📱 Telegram 配置

#### 获取 Bot Token

1. 打开 Telegram，搜索 `@BotFather`
2. 发送 `/newbot` 命令
3. 按提示设置机器人名称和用户名
4. 复制 Bot Token（格式：`1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`）

#### 获取 Chat ID

**方法 1：个人聊天**
1. 搜索你创建的机器人
2. 发送 `/start` 命令
3. 访问：`https://api.telegram.org/bot<你的Bot_Token>/getUpdates`
4. 找到 `"chat":{"id": 123456789}` 中的数字

**方法 2：群组聊天**
1. 将机器人添加到群组
2. 在群组中发送一条消息
3. 访问上面的 URL
4. 找到群组的 Chat ID（负数，如 `-123456789`）

#### 配置示例

```
Bot Token: 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
Chat ID: 123456789
```

#### 测试消息

配置完成后，点击"测试"，你的 Telegram 会收到：
```
🧪 测试消息

这是来自量化交易系统的测试消息。
如果你看到这条消息，说明机器人配置成功！
```

### 💬 Discord 配置

#### 创建 Webhook

1. 打开 Discord，进入你的服务器
2. 右键点击频道，选择 **"编辑频道"**
3. 点击 **"整合"** → **"Webhook"**
4. 点击 **"新建 Webhook"**
5. 设置名称和头像
6. 点击 **"复制 Webhook URL"**

#### 配置示例

```
Webhook URL: https://discord.com/api/webhooks/123456789/abcdefghijklmnopqrstuvwxyz
```

#### 预警消息效果

触发预警时，Discord 会收到富文本消息：
- **标题**：🔔 价格预警触发
- **颜色**：绿色（突破）或橙色（跌破）
- **内容**：包含交易对、价格、时间等信息

### 💼 Slack 配置

#### 创建 Webhook

1. 访问 [Slack API](https://api.slack.com/messaging/webhooks)
2. 点击 **"Create an app"**
3. 选择 **"From scratch"**
4. 输入应用名称，选择工作区
5. 进入 **"Incoming Webhooks"**
6. 启用 **"Activate Incoming Webhooks"**
7. 点击 **"Add New Webhook to Workspace"**
8. 选择要发送消息的频道
9. 复制 Webhook URL

#### 配置示例

```
Webhook URL: https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXX
```

#### 预警消息效果

触发预警时，Slack 会收到格式化消息：
- **标题**：🔔 价格预警触发
- **附件**：带颜色的消息卡片
- **时间戳**：自动添加

## 故障排除

### Telegram 无法发送

**问题 1：Bot Token 错误**
- 检查 Token 格式是否正确
- 确认 Token 是否有效
- 重新从 @BotFather 获取

**问题 2：Chat ID 错误**
- 确认 Chat ID 是数字
- 群组 Chat ID 应该是负数
- 使用 getUpdates API 重新获取

**问题 3：机器人未启动**
- 在 Telegram 中搜索机器人
- 发送 `/start` 命令
- 确认机器人已启动

### Discord 无法发送

**问题 1：Webhook URL 无效**
- 检查 URL 是否完整
- 确认 Webhook 未被删除
- 在 Discord 中重新创建

**问题 2：权限不足**
- 确认 Webhook 有发送消息权限
- 检查频道设置

### Slack 无法发送

**问题 1：Webhook 过期**
- Webhook 可能被撤销
- 重新创建 Webhook

**问题 2：应用未安装**
- 确认应用已安装到工作区
- 检查应用权限

### 企业微信无法发送

**问题 1：Webhook 地址错误**
- 检查地址是否完整
- 确认包含正确的 key

**问题 2：机器人被删除**
- 在群聊中重新添加机器人
- 获取新的 Webhook 地址

### 测试成功但实际不发送

**可能原因：**
1. 机器人被禁用（开关为灰色）
2. 浏览器网络问题
3. API 请求被阻止（CORS）

**解决方法：**
1. 确认机器人已启用
2. 检查浏览器控制台错误
3. 确认网络连接正常

## 安全与隐私

### 配置存储

- 机器人配置保存在浏览器本地存储
- 不会上传到服务器
- 刷新页面后自动加载

### Token 安全

- Token 输入框使用密码模式
- 不会在界面上明文显示
- 建议定期更换 Token

### 建议

1. **不要分享 Token**：任何人拥有 Token 都可以控制机器人
2. **定期检查**：定期检查机器人是否被滥用
3. **最小权限**：只授予必要的权限
4. **备份配置**：重要配置建议记录备份

## 常见问题

### Q: 支持多个机器人吗？
**A:** 是的，你可以添加多个不同平台的机器人，也可以添加同一平台的多个机器人。

### Q: 机器人会发送什么内容？
**A:** 只发送价格预警触发消息，包含交易对、目标价格、当前价格和时间。

### Q: 消息会延迟吗？
**A:** 几乎实时，通常在预警触发后 1-2 秒内发送到平台。

### Q: 会收费吗？
**A:** 本功能免费，但各平台的 API 调用限制由平台决定。Telegram、Discord、Slack 的基础使用都是免费的。

### Q: 配置会丢失吗？
**A:** 配置保存在浏览器本地存储，清除浏览器数据会导致配置丢失。建议记录备份。

### Q: 可以只通知某些预警吗？
**A:** 当前版本会通知所有触发的预警。未来版本将支持按预警类型或重要性筛选。

### Q: 支持图片通知吗？
**A:** 当前版本仅支持文本。未来版本计划支持图表截图。

## 开发指南

### 添加新平台支持

编辑 `src/components/AlertPanel.tsx`：

```typescript
// 添加平台信息
const platformInfo = {
  // ... 现有平台
  newplatform: {
    icon: '🆕',
    name: '新平台',
    fields: [
      { key: 'apiKey', label: 'API Key', placeholder: 'xxx', type: 'password' }
    ],
    guide: 'https://...'
  }
}

// 添加发送逻辑
const sendNewPlatformMessage = async (bot: BotConfig, message: string) => {
  await fetch(bot.config.apiUrl, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${bot.config.apiKey}` },
    body: JSON.stringify({ text: message })
  })
}
```
